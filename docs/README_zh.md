## ⚠️ 需要改進的地方（優先順序排序）

### 🔴 優先級 1：型別安全問題（急需修復）

**問題說明**：
在一些關鍵檔案中，程式碼使用了 `any` 型別，破壞了 TypeScript 的型別檢查。

**問題位置**：
- `src/services/journalService.ts` (5 處)
- `src/pages/Index.tsx` (1 處)

**範例**：
```typescript
// ❌ 當前寫法（不安全）
const mapToJournalEntry = (entry: any): JournalEntry => {
  const cards = (entry.journal_entry_cards as any[]).map(...);
}

// ✅ 應該改為（安全）
type JournalEntryWithCards = {
  ...Database['public']['Tables']['journal_entries']['Row'],
  journal_entry_cards?: Array<{ card_name: string }>;
};

const mapToJournalEntry = (entry: JournalEntryWithCards): JournalEntry => {
  const cards = entry.journal_entry_cards?.map(card => card.card_name) ?? [];
}
```

**為何重要**：
- 防止執行時錯誤
- 提升 IDE 自動補全能力
- 讓程式碼更容易維護

**修復時間**：2-3 小時

---

### 🔴 優先級 2：資料一致性問題（急需清理）

**問題說明**：
卡片資料有兩個儲存位置，可能導致資料不同步：
1. `journal_entries.cards` 欄位（text[] 陣列）- **遺留欄位，應移除**
2. `journal_entry_cards` 表（關聯表）- **正確的做法**

**為何重要**：
- 避免「雙重真相來源」（Single Source of Truth 原則）
- 簡化程式碼邏輯
- 降低資料不一致風險

**解決方案**：
1. 確認所有資料已在 `journal_entry_cards` 表中
2. 新增 SQL Migration 移除舊欄位：
   ```sql
   ALTER TABLE journal_entries DROP COLUMN IF EXISTS cards;
   ```
3. 更新 TypeScript 型別定義

**修復時間**：1 小時

---

### 🟡 優先級 3：Index.tsx 職責過重（中期改進）

**問題說明**：
`src/pages/Index.tsx` 檔案有 470 行程式碼，包含了太多職責：
- 資料查詢邏輯
- CRUD 操作
- 統計計算
- UI 渲染

**為何重要**：
- 單一檔案過大，難以維護
- 業務邏輯與 UI 混雜，難以測試
- 違反「單一職責原則」

**建議重構**：
```
提取為：
├─ src/services/journalAnalytics.ts    （統計邏輯）
├─ src/hooks/useJournalMutations.ts    （CRUD 操作）
└─ src/pages/Index.tsx                 （僅負責 UI）
```

**修復時間**：4-6 小時

### 🟡 優先級 4：錯誤處理不足（中期改進）

**問題說明**：
當 AI 服務出錯時，使用者看到的是原始錯誤訊息（例如：`Error: 429 Too Many Requests`），不夠友善。

**建議改善**：
```typescript
// ❌ 當前

catch (error) {
  if (error.message.includes('quota')) {
    throw new Error('AI 解讀額度已用盡，請明日再試 🔮');
  }
  // ... 其他錯誤分類
}
```

**修復時間**：2 小時
